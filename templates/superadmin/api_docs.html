Get Your Token

All API requests require a JWT Bearer token (except login and register).

Access Token (use in Authorization header)
Expires in 12 hours. Use refresh token to renew.
Quick Examples
Login
curl -X POST /api/auth/login/ \ -H "Content-Type: application/json" \ -d '{"username":"admin","password":"pass"}'
Get Rooms (with token)
curl /api/rooms/ \ -H "Authorization: Bearer <token>"
JavaScript Fetch
const res = await fetch('/api/rooms/', { headers: { 'Authorization': 'Bearer ' + token, 'Content-Type': 'application/json' } }); const data = await res.json();
Register (no password needed)
POST /api/auth/register/ { "username": "juan", "first_name": "Juan", "last_name": "Dela Cruz", "email": "juan@school.ph", "role": "student", "grade_level": "10" }

AUTH
POST/api/auth/login/Get JWT tokens — body: {username, password}
POST/api/auth/register/Register new account — body: {username, first_name, last_name, email, role, grade_level}
POST/api/auth/token/refresh/Refresh access token — body: {refresh}
GET/api/auth/me/Get current user info
PATCH/api/auth/me/Update profile — body: {first_name, last_name, email}
POST/api/auth/change-password/Change password — body: {old_password, new_password}
ROOMS
GET/api/rooms/List rooms for the authenticated user
GET/api/rooms/<id>/Room detail with teacher info and student count
GET/api/rooms/<id>/lessons/All lessons in a room
GET/api/rooms/<id>/materials/All materials (files and links) in a room
GET/api/rooms/<id>/assignments/Assignments with student submission status if applicable
GET/api/rooms/<id>/quizzes/Quizzes with attempt status
GET/api/rooms/<id>/live/Live class sessions (Jitsi Meet)
POST/api/rooms/<id>/live/Create live session (teacher only) — body: {title, scheduled_at, description}
ASSIGNMENTS & QUIZZES
GET/api/assignments/<id>/Assignment detail
POST/api/assignments/<id>/submit/Submit assignment (student) — multipart/form-data: {file, notes}
GET/api/quizzes/<id>/Quiz with questions (correct_answer hidden from students)
POST/api/quizzes/<id>/submit/Submit quiz answers — body: {"answers": {"<question_id>": "a"|"b"|"c"|"d"}}
GRADES & ANNOUNCEMENTS
GET/api/grades/Student's assignment grades and quiz attempts with summary stats
GET/api/announcements/Announcements for the user — ?type=urgent&unread=1&page=1
POST/api/announcements/<id>/read/Mark announcement as read
DASHBOARD & NOTIFICATIONS
GET/api/dashboard/Full dashboard summary (role-aware stats and recent rooms)
GET/api/notifications/Unread counts: announcements, pending assignments, live sessions
ADMIN ONLY
GET/api/admin/users/All users — ?role=student&status=pending&q=search
POST/api/admin/users/<id>/approve/Approve user and send password setup link email
GET/api/admin/stats/System-wide statistics (users, rooms, errors)
