1. System Overview

EduLMS is a web-based Learning Management System designed specifically for Philippine high school environments (Grades 7–12). The system organises learning by grade level and subject, ensuring each virtual "Room" corresponds to exactly one subject for one grade level.

Core principle: Teachers control all student access. Students cannot join or browse rooms independently — they must be assigned by a teacher. This prevents misplacement errors and maintains academic structure.

3. Registration & Approval Flow

Student Flow

  1. Student fills registration form (no password required)
  2. OTP sent to student's email — must verify within 10 minutes
  3. Account created with pending status
  4. Teacher approves the student account
  5. Student receives a unique password setup link (valid 72 hours)
  6. Student sets their own password and is auto-logged in

Teacher Flow

  1. Teacher fills registration form
  2. OTP verified (email confirmed)
  3. Super Admin approves the teacher account
  4. Teacher receives a unique password setup link (valid 72 hours)
  5. Teacher sets password and accesses dashboard

4. Room (Course) System

Each Room enforces two hard constraints at the database level:

  • One subject per room — e.g. Mathematics, Science, English
  • One grade level per room — Grades 7 through 12

Teachers assign students to rooms manually. Only students whose registered grade level matches the room's grade level appear in the assignment list — preventing cross-grade access.

7. Security Features

  • RBAC — Role-Based Access Control via decorators on every view
  • OTP Email Verification — 6-digit code, 10-minute expiry, 5-attempt limit
  • Password Setup Tokens — 64-byte URL-safe, single-use, 72-hour expiry
  • Tab Detection — Detects tab switching during quizzes (students only), logs violations
  • Error Detection Middleware — Auto-captures all unhandled exceptions with full context
  • CSRF Protection — Django CSRF on all forms; ngrok/production URLs whitelisted
